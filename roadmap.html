DevOps Learning Roadmaps

1. Version Control Introduction

Git is a distributed version control system used to track source code changes. It's crucial in DevOps for managing code, collaboration, automation, and CI/CD pipelines.

2. Installing Git


# Ubuntu
sudo apt update
sudo apt install git

# macOS
brew install git

# Configure Git
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

3. Git CLI Essentials


git init                # Initialize a repo
git clone          # Clone repo
git add .               # Stage all changes
git commit -m "message" # Commit changes
git push origin main    # Push to GitHub

4. Branching & Merging


git checkout -b dev             # Create new branch
git merge dev                   # Merge into current branch
git branch -d dev               # Delete branch

5. Git in DevOps

  • CI/CD: GitHub triggers Jenkins pipelines using webhooks
  • GitOps: Use Git to store infrastructure code (Terraform, K8s YAMLs)
  • Strategy: Follow Git Flow - develop, feature, staging, release, main
